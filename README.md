# ranch_ledger

On-chain cattle lifecycle ledger.

`BovineTracking` is a Solidity 0.8.28 contract that records vaccines,
movements, feed, health exams, and abattoir processing for each bovine.
Companion contracts `BovineNFT` (ERC-721) and `RanchToken` (ERC-20) anchor
provenance and reward actors in the supply chain. The off-chain Express +
MongoDB service bridges the on-chain records with a familiar CRUD API.

The project is built and tested with **Foundry** (forge / anvil / cast) and
the off-chain service uses **ethers v6**.

## Layout

```
src/                  Solidity sources (0.8.28, optimizer on, cancun)
  BovineTracking.sol  Access-controlled lifecycle ledger
  BovineNFT.sol       ERC-721 - one NFT per bovine
  RanchToken.sol      ERC-20 - rewards / utility token
test/                 Solidity unit + fuzz tests (forge test)
script/               Forge scripts (deploy, bulk mint)
lib/                  forge-std + OpenZeppelin v5.1
services/             ethers v6 wrapper for the Express server
deployments/          Generated by Deploy.s.sol (gitignored)
server.js             Express + MongoDB API
foundry.toml          Foundry config
remappings.txt        OZ + forge-std remappings
```

## Quick start (local anvil)

```bash
# 1. start a local anvil with 100 prefunded accounts
npm run anvil

# 2. in another shell, build + test
npm run build
npm test

# 3. deploy the full stack
npm run deploy
# writes deployments/local.json with the 3 contract addresses

# 4. spawn 100 agents that each register one bovine
npm run bulk-mint

# 5. start the off-chain API
PRIVATE_KEY=0xac0974bec39a17e36ba4a6b4d238ff944bacb478cbed5efcae784d7bf4f2ff80 \
  MONGODB_URI=mongodb://localhost:27017/bovine-tracker \
  node server.js
```

`npm run dev` runs anvil + the server in the same terminal via
`concurrently`.

## Smart contract surface

### Roles (AccessControl)

| Role           | Who                         | Can do                          |
| -------------- | --------------------------- | ------------------------------- |
| `DEFAULT_ADMIN_ROLE` | Deployer / multisig    | Grant + revoke other roles      |
| `REGISTRAR_ROLE`     | Farm / cooperative     | `addBovine`                     |
| `VET_ROLE`           | Veterinarians          | `addVaccine`, `addHealthExam`   |
| `RANCHER_ROLE`       | Ranch hands            | `addMovement`, `addFeed`        |
| `ABBATTOIR_ROLE`     | Slaughterhouses        | `addAbattoirProcess`            |

### Errors

`InvalidBovine(uint256)`, `DuplicateBovineName(string)`,
`EmptyString(string)`, `InvalidAge(uint256)`,
`NoNFTReceiver()` (from `BovineNFT`).

### Events

`BovineAdded`, `VaccineAdded`, `MovementAdded`, `FeedAdded`,
`HealthExamAdded`, `AbattoirProcessAdded`, plus OZ `RoleGranted` /
`RoleRevoked` and ERC-721 `Transfer`.

## Tests

```bash
npm test                # 17 tests (3 suites) - includes a fuzz test
npm run test:gas        # gas report
```

The Solidity test suite supersedes the old JS-based `bovine_tracking.js` and
covers the same scenarios (add bovine, add vaccine, add feed, add movements,
add abattoir processes, add health exams, aggregate read, custom-error
reverts, role enforcement, fuzz `addBovine`).

## Deploying to a public network

```bash
PRIVATE_KEY=<hex> \
RPC_URL=https://sepolia.infura.io/v3/<id> \
forge script script/Deploy.s.sol --rpc-url $RPC_URL --broadcast --verify
```

`ADMIN` defaults to `0xf39Fd6...92266` (anvil[0]). Override with
`ADMIN=<checksummed address> forge script ...`.

## Architecture

```
+-------------------+        +------------------+
|  Express server   | <----> |  ethers v6 RPC   |
| (server.js)       |        |  (anvil / node)  |
+-------------------+        +------------------+
        |                             |
        v                             v
+-------------------+        +------------------+
|    MongoDB        |        | BovineTracking   | <-- BovineNFT
| (off-chain index) |        | RanchToken       | <-- OpenZeppelin
+-------------------+        +------------------+
```

The on-chain contract is the source of truth. MongoDB stores the same data
for fast querying and to back the REST API; the service rolls back the Mongo
write if the on-chain transaction fails.

## 100-agent simulation

`script/BulkMint.s.sol` is a Foundry script that spawns 100 deterministic anvil-funded agents, grants each `REGISTRAR_ROLE`, and has each register a unique bovine (`Bessie-0`, `Daisy-1`, `Molly-2`, …). On a fresh anvil with 1s block time the full 300-transaction batch takes ~5 minutes end-to-end.

**Last verified:** 100 bovines on local anvil (chain 31337), 20 per breed × 5 breeds, each owned by a distinct agent address. See `docs/ARCHITECTURE.md` §6 for the design rationale.
