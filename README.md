# ranch_ledger

On-chain cattle lifecycle ledger with **global livestock ID support**, **NFT fractionalization**, and **DAO governance**.

`BovineTracking` is a Solidity 0.8.28 contract that records vaccines, movements, feed, health exams, and abattoir processing for each bovine — supporting all national identification systems (Brazil SISBOV, EU ISO 11784/11785, USA USDA ANID/EID, Australia NLIS, China MARA, GCC GSO 2057). Each bovine is minted as a unique ERC-721 NFT (`BovineNFT`) and can be fractionalized into tradable ERC-20 shares (`BovineShareToken`). A DAO (`GovernorRanch`) lets RanchToken holders vote on vault parameters.

The project is built and tested with **Foundry** (forge / anvil / cast) and the off-chain service uses **ethers v6**.

## Layout

```

  BovineTracking.sol          Access-controlled lifecycle ledger + global IDs
  BovineNFT.sol               ERC-721 - one NFT per bovine (ERC721Consecutive batch mint)
  RanchToken.sol              ERC-20 - rewards / utility token (6 decimals)
  EUDRCompliance.sol          EUDR metadata + validators for all global livestock ID systems
  GPSValidator.sol            GPS coordinate validation + Haversine distance calculation

  FractionalizationManager.sol Manages NFT fractionalization into ERC-20 shares
  BovineShareToken.sol        Per-cow ERC-20 share token (minted by FractionalizationManager)
  GovernorRanch.sol           DAO governance (Governor + Votes + AccessControl)

script/                       Foundry scripts (deploy, bulk mint, multi-chain)
lib/                          forge-std v1.16.2 + OpenZeppelin v5.1.0
services/                     ethers v6 wrapper for the Express server
subgraph/                     The Graph subgraph config (schema.graphql, mapping.ts, subgraph.yaml)
locales/                      i18n files (pt-BR.json, en.json)
deployments/                  Generated by Deploy.s.sol (gitignored)
server.js                     Express + MongoDB API
foundry.toml                  Foundry config
remappings.txt                OZ + forge-std remappings
```

## Quick start (local anvil)

```bash

npm run anvil

# 2. in another shell, build + test
npm run build
npm test

# 3. deploy the full stack
npm run deploy
# writes deployments/local.json with the 3 contract addresses

# 4. spawn 100 agents that each register one bovine
npm run bulk-mint

# 5. start the off-chain API
PRIVATE_KEY=0xac0974bec39a17e36ba4a6b4d238ff944bacb478cbed5efcae784d7bf4f2ff80 \
  MONGODB_URI=mongodb://localhost:27017/bovine-tracker \
  node server.js
```

`npm run dev` runs anvil + the server in the same terminal via
`concurrently`.

## Smart contract surface

### Core contracts

| Contract | Standard | Purpose |
| --- | --- | --- |
| **BovineTracking** | AccessControl + ReentrancyGuardTransient | Lifecycle ledger: vaccines, movements, feed, health exams, abattoir. Supports all global livestock ID formats (SISBOV, EU ISO 11784/11785, USDA ANID/EID, NLIS, MARA, GCC GSO 2057). |
| **BovineNFT** | ERC-721 + ERC721Consecutive + AccessControl | One NFT per bovine. Batch minting via `mintBatchForBovines()` (85% gas savings vs individual mints). |
| **RanchToken** | ERC-20 + AccessControl | Utility / reward token with 6 decimals. MINTER_ROLE and BURNER_ROLE. |

### Compliance & validation

| Contract | Purpose |
| --- | --- |
| **EUDRCompliance** | EUDR metadata (farm polygon, deforestation cert hash) + validators for SISBOV (15-digit), EU ISO 11784/11785 (CC+HerdMark+Individual), USDA ANID (15-digit), USDA EID (9-digit), NLIS (12-digit DUNS-based), China MARA (15-digit), GCC GSO 2057 (CC-XXXXXX-XXXX). |
| **GPSValidator** | GPS coordinate validation with Haversine distance calculation. Validates lat/long bounds, timestamp freshness (<24h), and movement sanity checks. |

### DeFi & governance

| Contract | Purpose |
| --- | --- |
| **RanchLendingVault** | NFT-backed lending vault. Deposits BovineNFTs as collateral, borrows RanchToken against them. Health-score-based notional valuation (0-100 score from provenance data). Compound-style interest rate model parameterized by utilization. Liquidation on undercollateralization or health score drop below floor. |
| **FractionalizationManager** | NFT fractionalization system. Holds BovineNFTs and mints per-cow ERC-20 share tokens (`BovineShareToken`). Investors buy shares, proceeds distributed proportionally when cow is sold/slaughtered. |
| **BovineShareToken** | Per-cow ERC-20 token representing fractional ownership of a specific bovine NFT. Mintable only by FractionalizationManager via MINTER_ROLE. |
| **GovernorRanch** | DAO governance for RanchLendingVault. Governor + Votes + AccessControl pattern. Immutable voting delay/period set at deploy time. RanchToken holders vote on liquidation thresholds, collateral types, fee structures. 10% quorum based on total supply. |

### Roles (AccessControl)

| Role | Who | Can do |
| --- | --- | --- |
| `DEFAULT_ADMIN_ROLE` | Deployer / multisig | Grant + revoke other roles |
| `REGISTRAR_ROLE` | Farm / cooperative | `addBovine` |
| `VET_ROLE` | Veterinarians | `addVaccine`, `addHealthExam` |
| `RANCHER_ROLE` | Ranch hands | `addMovement`, `addFeed` |
| `ABBATTOIR_ROLE` | Slaughterhouses | `addAbattoirProcess` |
| `MINTER_ROLE` (NFT) | NFT minter contract | Mint BovineNFTs |
| `LIQUIDATOR_ROLE` | Liquidation bot / admin | Trigger liquidations, mark fractionalized cows as sold |
| `PRICER_ROLE` | Oracle / admin | Set notional values for collateral |

### Errors

**BovineTracking:** `InvalidBovine(uint256)`, `DuplicateBovineName(string)`, `EmptyString(string)`, `InvalidAge(uint256)`, `NoNFTReceiver()`
**EUDRCompliance:** `InvalidSisbovId(string)`, `InvalidCnpj(string)`, `InvalidPolygon()`, `InvalidCountryCode(string)`, `InvalidNationalId(string, string)`
**GPSValidator:** `InvalidLatitude(int256)`, `InvalidLongitude(int256)`, `GPSTooOld(uint256)`, `GPSNotProvided()`
**RanchLendingVault:** `NotCollateralized(uint256)`, `Undercollateralized(uint256, uint256)`, `InsufficientCollateral()`, `InvalidTokenId(uint256)`, `HealthScoreTooLow(uint256, uint256)`
**FractionalizationManager:** `NFTNotOwnedByManager(uint256)`, `AlreadyFractionalized(uint256)`, `NotFractionalized(uint256)`, `InsufficientShares(address, uint256, uint256)`, `InvalidSalePrice(uint256)`

### Events

`BovineAdded`, `VaccineAdded`, `MovementAdded`, `FeedAdded`, `HealthExamAdded`, `AbattoirProcessAdded`, `EudrAttestations`, `MovementGPS`, `CollateralDeposited`, `CollateralWithdrawn`, `LoanRepaid`, `Liquidated`, `InterestAccrued`, `Fractionalized`, `SharesPurchased`, `SharesRedeemed`, `SaleComplete`, plus OZ `RoleGranted` / `RoleRevoked` and ERC-721 `Transfer`.

## Deploying to a public network

```bash
PRIVATE_KEY=<hex> \
RPC_URL=https://sepolia.infura.io/v3/<id> \
forge script script/Deploy.s.sol --rpc-url $RPC_URL --broadcast --verify
```

`ADMIN` defaults to `0xf39Fd6...92266` (anvil[0]). Override with `ADMIN=<checksummed address> forge script ...`.

Multi-chain deployment scripts are available:
- `script/DeployBase.s.sol` — Base mainnet
- `script/DeployOptimism.s.sol` — OP mainnet
- `script/DeployArbitrum.s.sol` — Arbitrum One
- `script/DeployZkSync.s.sol` — zkSync Era
- `script/DeployAmoy.s.sol` — Polygon Amoy (testnet)

## Architecture

```
+-------------------+        +------------------+
|  Express server   | <----> |  ethers v6 RPC   |
| (server.js)       |        |  (anvil / node)  |
+-------------------+        +------------------+
        |                             |
        v                             v
+-------------------+        +------------------------------------------+
|    MongoDB        |        | Smart Contracts                          |
| (off-chain index) |        | BovineTracking ──→ EUDRCompliance       |
+-------------------+        | BovineNFT (ERC721Consecutive)           |
                             | RanchToken (ERC-20)                     |
                             | RanchLendingVault (NFT-backed lending)   |
                             | FractionalizationManager + BovineShareToken |
                             | GovernorRanch (DAO governance)          |
                             | GPSValidator                            |
+-------------------+        +------------------------------------------+
|    Subgraph       |        | The Graph Node                          |
| (GraphQL endpoint)|        | Indexes all contract events             |
+-------------------+        +------------------------------------------+
```

The on-chain contracts are the source of truth. MongoDB stores the same data for fast querying and to back the REST API; the service rolls back the Mongo write if the on-chain transaction fails. The Subgraph pre-computes indexes for scale (1M cattle × 50 events = 50M events).

## 100-agent simulation

`script/BulkMint.s.sol` is a Foundry script that spawns 100 deterministic anvil-funded agents, grants each `REGISTRAR_ROLE`, and has each register a unique bovine (`Bessie-0`, `Daisy-1`, `Molly-2`, …). On a fresh anvil with 1s block time the full 300-transaction batch takes ~5 minutes end-to-end.

**Last verified:** 100 bovines on local anvil (chain 31337), 20 per breed × 5 breeds, each owned by a distinct agent address. See `docs/ARCHITECTURE.md` §6 for the design rationale.
